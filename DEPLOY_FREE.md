# Развёртывание BIBLIOTECH

## Временный доступ из интернета

Для проверки локального сервера с другого устройства можно использовать Cloudflare Quick Tunnel.

Сначала запустите проект:

```powershell
npm install
npm start
```

Затем откройте туннель к локальному порту:

```powershell
cloudflared tunnel --url http://localhost:3001
```

Команда выдаст временный HTTPS-адрес. Он работает только пока запущены компьютер, Node.js-сервер и `cloudflared`.

## Публикация на Render

В корне проекта находится `render.yaml` с настройками веб-сервиса и PostgreSQL.

1. Добавьте репозиторий в свой аккаунт Render.
2. В панели Render выберите **New → Blueprint**.
3. Укажите репозиторий BIBLIOTECH.
4. Подтвердите создание веб-сервиса и базы PostgreSQL.
5. Дождитесь завершения сборки и проверьте адрес, выданный Render.

Бесплатный тариф подходит для демонстрации и учебной работы. После периода бездействия первый запрос может выполняться дольше обычного. Условия тарифов Render со временем могут меняться.

## Переменные окружения

Файл `render.yaml` настраивает:

- `NODE_ENV=production`;
- случайное значение `JWT_SECRET`;
- `DATABASE_URL` из созданной базы PostgreSQL.

Для каскадного поиска книг по ISBN можно дополнительно задать:

- `GOOGLE_BOOKS_API_KEY` — ключ Google Books API;

Ключ используется только сервером и не передаётся в браузер. Поиск сначала проверяет Open Library, затем Google Books, а при отсутствии результата предлагает ручное заполнение.

При запуске сервер применяет `backend/sql/init.sql`, создаёт недостающие таблицы и добавляет тестовую учётную запись администратора.

Тестовый вход:

```text
Логин: admin
Пароль: GreenScreen
```

Перед публичной публикацией пароль администратора следует заменить.

## Проверка после публикации

Откройте служебный адрес:

```text
https://ВАШ-ДОМЕН/api/health
```

При исправной работе сервер вернёт JSON со статусом `OK`.
