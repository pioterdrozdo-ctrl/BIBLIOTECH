# BIBLIOTECH

BIBLIOTECH — веб-приложение для учёта библиотечного фонда. В нём можно вести каталог книг, учитывать количество экземпляров, оформлять выдачи и бронирования, хранить комментарии и находить книги по QR-коду или месту хранения.

Серверная часть написана на Node.js и Express. Она раздаёт интерфейс из папки `frontend` и обслуживает API. Основное хранилище — PostgreSQL; при локальном запуске без базы используется JSON-файл.

## Основные возможности

- регистрация, вход и гостевой режим;
- каталог книг с поиском, фильтрами и сортировкой;
- карточки книг с обложкой, описанием и количеством экземпляров;
- выдача, возврат и бронирование книг;
- комментарии и статистика фонда;
- QR-коды для быстрого открытия карточек;
- указание стеллажа и места хранения;
- карта кабинета 125.

Карта помещения пока работает в бета-режиме. Координаты объектов используются как навигационная схема и могут уточняться администратором.

## Запуск

Требуется Node.js 18 или новее.

```bash
npm install
npm start
```

После запуска сайт доступен по адресу:

```text
http://localhost:3001
```

Проверить состояние сервера можно командой:

```bash
npm run health
```

## Хранилище данных

При наличии `DATABASE_URL` сервер подключается к PostgreSQL и применяет схему из `backend/sql/init.sql`.

Без настроенной базы используется локальное JSON-хранилище. Такой режим подходит для разработки и проверки интерфейса, но не рассчитан на постоянную работу нескольких пользователей.

## Структура проекта

- `frontend/` — страницы, стили, клиентские скрипты и изображения;
- `backend/server.js` — настройка и запуск Express-сервера;
- `backend/routes/` — маршруты API;
- `backend/services/` — работа с локальным хранилищем и служебная логика;
- `backend/sql/init.sql` — структура PostgreSQL и начальные данные;
- `android-app/` — оболочка Trusted Web Activity для Android;
- `render.yaml` — конфигурация развёртывания на Render;
- `DEPLOY_FREE.md` — порядок публикации проекта.

## Развёртывание

Для Render в репозитории находится файл `render.yaml`. Он создаёт веб-сервис и PostgreSQL, после чего передаёт приложению строку подключения через `DATABASE_URL`.

Подробные шаги приведены в `DEPLOY_FREE.md`.
